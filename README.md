# Spotify Track Popularity Prediction

Predicting Spotify track popularity (0–100) from audio features using LightGBM, Random Forest, and Linear Regression. Includes SHAP explainability analysis and genre mean-target encoding.

## Project structure

```
spotify-popularity-prediction/
├── data/
│   ├── raw/                  # Place downloaded dataset here
│   └── processed/            # Auto-generated by notebook 02
├── notebooks/
│   ├── 01_eda.ipynb          # Exploratory data analysis
│   ├── 02_preprocessing.ipynb# Cleaning, encoding, splitting, scaling
│   ├── 03_modeling.ipynb     # Train & evaluate all 4 models
│   └── 04_shap.ipynb         # SHAP analysis + ablation study
├── src/
│   ├── preprocessing.py      # Reusable preprocessing functions
│   ├── models.py             # Model definitions and training helpers
│   └── evaluation.py         # Metrics and plotting utilities
├── models/                   # Saved model artifacts (.pkl / .pt)
├── figures/                  # All saved plots for the paper
├── requirements.txt
└── README.md
```

## Setup

```bash
git clone https://github.com/yourusername/spotify-popularity-prediction.git
cd spotify-popularity-prediction
pip install -r requirements.txt
```

## Data

Download the [Spotify Tracks Dataset](https://www.kaggle.com/datasets/maharshipandya/-spotify-tracks-dataset) from Kaggle and place `dataset.csv` in `data/raw/`.

## Running the project

Run the notebooks in order:

```bash
jupyter notebook notebooks/01_eda.ipynb
jupyter notebook notebooks/02_preprocessing.ipynb
jupyter notebook notebooks/03_modeling.ipynb
jupyter notebook notebooks/04_shap.ipynb
```

Each notebook saves its outputs (processed data, models, figures) so the next notebook can load them directly.
